# 待测试

- 配置与用户偏好：每个模型渠道新增 OpenAI / Gemini 调用格式选择，默认 OpenAI，切换默认格式时同步更新默认 Base URL；渠道 Tab 增加紧凑醒目的提醒，说明模型是否可选需要到“模型”Tab 选择。
- Gemini 调用格式：支持拉取 Gemini 模型列表，并用于文本问答、在线 Agent 工具调用、基础生图和图生图请求。
- 音频和视频生成：选择 Gemini 调用格式时先给出不支持提示，仍需使用 OpenAI 兼容渠道。
- 修复 Grsai 渠道流式对话 404 的问题：原逻辑会自动将 `grsai.com` / `grsai.ai` 域名强行替换为失效域名 `grsaiapi.com`，导致 API 接口访问 404 网页。现已移除该替换规则，保持用户填写的原始域名。
- API 错误排查增强：在 API fetch 请求失败的异常信息中附带了具体请求的 URL。请复现 404 错误并查看界面展示的详细报错 URL，以便最终定位请求地址。
- Base URL 鲁棒性兼容：若用户在系统设置的 Base URL 中误填了完整的 `/chat/completions` 后缀，程序将自动在发起请求前（无论是 OpenAI 格式还是 Gemini 格式）将该后缀洗掉，避免拼接出杂乱的 404 URL。
- 多模态图片自动压缩优化：在通过助手面板、画图流程等多模态方式向大模型发送含有大图（如 2K 原始图等 base64 数据）的请求前，系统将自动利用 Canvas 对 Base64 格式的大图进行等比例缩小（限制最大边长 1024px 并转为 80% 质量 JPEG 格式），以彻底根治大图发送时引发的 `Failed to fetch` 跨域与包体限制问题，并提升连接性能。
- 增强参考图转换异常输出：在将外部参考图转换为 Data URL 的 fetch 逻辑中加入详细的 try-catch，如果因跨域 (CORS)、网络错误或资源缺失引发 Failed to fetch，会附带输出具体的资源地址（url / storageKey），便于进一步排查参考图来源。
